Forms

The HTML <form> element defines a form that is used to collect user input:

An HTML form contains form elements.

The <input> element is the most important form element.

Input Types
Type Description
<input type="text"> Defines a one-line input field
<input type="radio"> Defines a radio button
<input type="submit"> Defines a submit button

Submit Button

a submit input type defines a button for submitting the form data to a form-handler

The form-handler is typically a server page with a script for processing input data.

The form-handler is specified in the form's action attribute:


The Action Attribute

The action attribute defines the action to be performed when the form is submitted.

Form data is sent to a web page on the server when the user clickes on the submit button

Example <form action="/action_page.php">

If the action attribute is omitted, the action is set to the current page.

The Target Attribute

The Target attribute specifies if the submitted result will open in a new browser tab, a frame or in the current window.

The default value is "_self"

To make the form result open in a new browser tab, use the value "_blank"

Get VS. Post

The method attribute specifies the HTTP method (get or post) to be used when submitting the form data

The default method when submitting for data is GET

Get makes form data visible in the page adress field

Never use Get to send sensitive data!

Get is better for non-secure data

Always use post when the form contains sensitive data or personal information


The Name Attribute

Each input field must have a name attribute to be submitted

If the name is omitted, the data of that input will not be sent



Grouping Form Data

The <fieldset> element is used to group related data in a form.

The <legend> element defines a caption for the fieldset element
